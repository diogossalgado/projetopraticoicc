Controle de Versões com Git

  • Página de Áudio
  • Página de Vídeo

O Git é um sistema de controle de versões distribuído amplamente utilizado para rastrear as alterações em arquivos ao longo do tempo, ele permite que várias pessoas trabalhem em um mesmo projeto de forma colaborativa, mantendo um histórico detalhado de todas as alterações feitas em cada arquivo. Cada usuário tem uma cópia completa do repositório, o que facilita o trabalho offline e torna o Git extremamente rápido e eficiente, mesmo para projetos grandes.

Com o Git, é possível ramificar o projeto em diferentes linhas de desenvolvimento, experimentar novas funcionalidades sem comprometer o código principal e facilmente mesclar as alterações de volta ao ramo principal quando estiverem prontas. Além disso, o Git oferece recursos avançados de gerenciamento de conflitos e recuperação de versões anteriores, o que garante a integridade e a segurança dos dados do projeto.

Comandos Git

Comando Descrição
git init Inicializa um repositório Git em um diretório.
git clone Clona um repositório Git existente para o seu sistema local.
git branch List, cria ou exclui branches.
git checkout Move o HEAD para um branch ou ponto específico na história.
git status Exibe o status das alterações no repositório.
git diff Exibe as diferenças entre arquivos no repositório.
git add Adiciona alterações ao índice para prepará-las para o commit.
git commit Registra as alterações no repositório.
git push Envia alterações locais para um repositório remoto.
git pull Atualiza o repositório local com as alterações do repositório remoto.
git revert Desfaz uma ou mais alterações em um commit específico.
git merge Combina o histórico de dois branches diferentes.
git stash Guarda temporariamente alterações não finalizadas.
git rm Remove arquivos do controle de versão.
git config Configuração do Git, como nome de usuário e email.
git reset Desfaz alterações no índice ou no diretório de trabalho para um commit específico.
git remote Gerencia repositórios remotos.
git fetch Busca alterações de um repositório remoto.
git show Exibe informações sobre um commit específico ou objeto.
git help Exibe a documentação de ajuda para um comando específico.
git rebase Aplica commits de um branch em outro.
git blame Exibe a última alteração em cada linha de um arquivo.
git tag Marca pontos específicos na história do repositório.

Configuração e acesso ao repositório Git:

  • Criando um novo repositório:

    • Navegue até o diretório do seu projeto local onde deseja iniciar o repositório Git.
    • Inicialize um novo repositório Git localmente com o seguinte comando:
    • git init
    • Isso criará um novo diretório `.git` localmente, que é onde o Git armazena seus metadados e configurações para o repositório.

  • Adicionar arquivos ao repositório:

    • Coloque seus arquivos no diretório do projeto que você iniciou.
    • Adicione os arquivos ao stage do Git usando o comando git add seguido do nome do arquivo ou git add . para adicionar todos os arquivos modificados e novos.
    • Exemplo: git add arquivo.txt

  • Confirmar e enviar para o repositório:

    • Para registrar as alterações feitas nos arquivos, execute o comando git commit -m "mensagem do commit" para confirmar as alterações.
    • Exemplo: git commit -m "Primeiro commit"
    • No GitHub, crie um novo repositório. Copie o URL do repositório que você acabou de criar.
    • Conecte seu repositório local ao repositório remoto no GitHub usando o comando git remote add origin URL_REPOSITORIO.
    • Exemplo: git remote add origin https://github.com/username/nomerepositorio.git
    • Finalmente, envie suas alterações para o repositório remoto usando o comando git push -u origin master.
    • Exemplo: git push -u origin master
    • Isso enviará os arquivos do seu repositório local para o repositório remoto no GitHub.

Utilização do Github na construção desse projeto:

Utilizei o GitHub no projeto para criar e enviar esta página HTML. Após finalizar o desenvolvimento, fiz um commit dos arquivos modificados e usei o comando git push para enviar as alterações para o repositório remoto.
